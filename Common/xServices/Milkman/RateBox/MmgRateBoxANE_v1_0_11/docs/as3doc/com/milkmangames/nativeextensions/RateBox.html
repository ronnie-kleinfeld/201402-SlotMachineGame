create()method 
public static function create(iosAppId:String, title:String, message:String, rateNowLabel:String = Rate Now, declineLabel:String = Not now, neverAgainlabel:String = Don't ask again, minLaunchesTilPrompt:int = 3, minEventsTilPrompt:int = 0, minDaysTilPrompt:int = 0, coolDownDays:int = 1):RateBox

Initializes the RateBox extension.

Parameters

iosAppId:String — the numeric iOS App ID. Find this on the app's page in iTunes Connect. If you're not supporting iOS, you can put an empty string ( "" ).
 
title:String — the title of the dialog displayed to the user for rating
 
message:String — the message to display to the user in the dialog
 
rateNowLabel:String (default = Rate Now) — label of the button the user pushes to rate the app
 
declineLabel:String (default = Not now) — label of the button the user pushes to decline rating the app at this time
 
neverAgainlabel:String (default = Don't ask again) — label of the button the user pushes to never see the rating prompt again
 
minLaunchesTilPrompt:int (default = 3) — the minimum number of times the app must be launched before the rating dialog box can be automatically displayed
 
minEventsTilPrompt:int (default = 0) — the minimum number of times a custom event must be triggered before the rating dialog box can be automatically displayed
 
minDaysTilPrompt:int (default = 0) — the minimum number of days for which the app must be installed before the rating dialog box can be displayed
 
coolDownDays:int (default = 1) — if the user presses 'not now' on a rating prompt dialog, this is the number of days that will pass before the prompt can be shown again.

Returns
RateBox — singleton instance of the RateBox object

Throws
Error — if RateBox.create() has already been called.

incrementEventCount()method 
public function incrementEventCount():Boolean

Increment the custom event count. A custom event can be anything you want; such as completing a level in the game. When the event count has reached at least the value for minEventsTilLaunch used when initializing the extension, the rating dialog may be displayed.

Returns
Boolean — true if the incrementEventCount() call has caused the rating dialog box to be displayed.

onLaunch()method 
public function onLaunch():Boolean

Call this method when your app is first started, or when it returns from a background state, to increment the launches counter. If the the counter has reached at least the value for minLaunchesTilPrompt with which the RateBox instance was created, the rating dialog box will be displayed.

Returns
Boolean — true if the call to onLaunch() caused the rating dialog box to be displayed
resetConditions()method 
public function resetConditions():void

Resets the rating conditions; the internal count of number of launches, number of events triggered, and days since first launch will be reset to 0. Also, if the user has previously selected to 'never see the rating dialog again', the preference will be cleared.

setAutoPrompt()method 
public function setAutoPrompt(useAutoPrompt:Boolean = true):void

Sets whether the rating prompt can be automatically displayed as a result of rating conditions changing. If you want to handle the display of the rating prompt manually, you can call setAutoPrompt(false), and use areRatingConditionsMet() to determine whether to display the prompt at a given time.

Parameters

useAutoPrompt:Boolean (default = true) — true for rating prompts to display automatically in response to onLaunch() or incrementEventCount(); false otherwise

useAmazonAppStore()method 
public function useAmazonAppStore():void

If you are deploying to the Amazon App Store, call this method to force the ratings page to load from the Amazon App Store instead of Google Play. (RateBox already uses the Amazon app store automatically when deployed on a Kindle Fire, but this method is useful if you are supporting the Amazon App Store for Google Android as well.)

useTestMode()method 
public function useTestMode():void

Call this function to activate test mode when testing the extension prior to app store release. In test mode, the user selecting the 'Rate' button will redirect them to an existing sample app on the app store, instead of your own app ID. Be sure to remove calls to useTestMode() before publishing your application.
