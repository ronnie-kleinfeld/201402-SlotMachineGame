scheduleLocalNotification()method 
public function scheduleLocalNotification(id:int, timeSeconds:Number, title:String, message:String, alertLabel:String = null, extraData:Object = null):void

Schedule a local notification to occur at a time in the future.

The id property should be a unique ID for this notification. You can use this ID to cancel the notificaiton explicitly in the future with the cancelLocalNotification method.

The timeSeconds parameter defines a time, in seconds, when this notification should occur. You can use the futureTimeSeconds method to easily get this value. For instance, CoreMobile.futureTimeSeconds(CoreMobile.WEEK) will return a time one week in the future.

The title and message properties determine the title and message on the notification. The alertLabel property will be used to set the action button on the notification popup. This is only used on iOS, in places such as the lock screen.

The extraData parameter allows you to set an object hash of key value pairs, that you can extract later to optionally attach game specific data to this notification event's extra property.

If the application is running in the foreground when the notification is triggered, CMLocalNotificationEvent.LOCAL_NOTIFICATION_RECEIVED will be dispatched. If the application is not running, or in the background, CMLocalNotificationEvent.RESUMED_FROM_LOCAL_NOTIFICATION will be dispatched after the user clicks the notification and causes the app to open.

Parameters

id:int — a unique integer id for this notificaiton
 
timeSeconds:Number — the date, in seconds, when this notification will occur
 
title:String — title of the notification
 
message:String — message of the notificaiton
 
alertLabel:String (default = null) — optional label for the action button on iOS notifications
 
extraData:Object (default = null) — optional hash of key value pairs to associate with this notification

scheduleRepeatingLocalNotification()method 
public function scheduleRepeatingLocalNotification(id:int, timeSeconds:Number, thenEverySeconds:Number, title:String, message:String, alertLabel:String = null, extraData:Object = null):void

Schedule a local notification to occur at a time in the future, then be repeated at a given interval until canceled.

The id property should be a unique ID for this notification. You can use this ID to cancel the notificaiton explicitly in the future with the cancelLocalNotification method.

The timeSeconds parameter defines a time, in seconds, when this notification should occur. You can use the futureTimeSeconds method to easily get this value. For instance, CoreMobile.futureTimeSeconds(CoreMobile.WEEK) will return a time one week in the future.

The thenEverySeconds parameter defines the interval, i nseconds, at which this notification will be repeated. Note that the interval may not be exact- the operating system may round this to a near regular interval, or adjust it to coincide with the appearence of other notifications on a close interval. You should use the constants such as CoreMobile.WEEK to set this interval.

The title and message properties determine the title and message on the notification. The alertLabel property will be used to set the action button on the notification popup. This is only used on iOS, in places such as the lock screen.

The extraData parameter allows you to set an object hash of key value pairs, that you can extract later to optionally attach game specific data to this notification event's extra property.

If the application is running in the foreground when the notification is triggered, CMLocalNotificationEvent.LOCAL_NOTIFICATION_RECEIVED will be dispatched. If the application is not running, or in the background, CMLocalNotificationEvent.RESUMED_FROM_LOCAL_NOTIFICATION will be dispatched after the user clicks the notification and causes the app to open.

Parameters

id:int — a unique integer id for this notificaiton
 
timeSeconds:Number — the date, in seconds, when this notification will occur
 
thenEverySeconds:Number — the interval, in seconds, at which this notification will be repeated
 
title:String — title of the notification
 
message:String — message of the notificaiton
 
alertLabel:String (default = null) — optional label for the action button on iOS notifications
 
extraData:Object (default = null) — optional hash of key value pairs to associate with this notification

showModalCredentialsDialog()method 
public function showModalCredentialsDialog(title:String, message:String, confirmButtonLabel:String, cancelButtonLabel:String = null, textInputType:int):CMDialogDispatcher

Show a modal native dialog that prompts the user to enter a name and password. The password field will be displayed as stars.

This function returns a CMDialogDispatcher object. You can listen for an event when the dialog is dismissed by calling .addDismissListener(listener) on the return value. CMDialogDispatcher dispatches an event of the type CMDialogEvent, with the details of the dialog and what the user selected. The event's modalUserName property will contain the name entered by the user, and its modalPassword will contain the password entered.

Parameters

message:String — message for this dialog
 
confirmButtonLabel:String — label for this dialog's button
 
cancelButtonLabel:String (default = null) — optionally, the label of a cancel button for this dialog
 
textInputType:int (default = NaN) — of CMInputType.TEXT, CMInputType.EMAIL or CMInputType.NUMBER indicating the keyboard style of the input text box

Returns
CMDialogDispatcher — a CMDialogDispatcher that will dispatch CMDialogEvent when the dialog is closed

startMonitoringDeviceMotion()method 
public function startMonitoringDeviceMotion():void

Start monitoring device motion (gyroscope data) on the current device. You should check isDeviceMotionAvailable() first to ensure the current device supports this feature.

While monitoring is enabled, you can read the current device motion state by calling getDeviceMotionData().

To stop monitoring device motion, call stopMonitoringDeviceMotion().
