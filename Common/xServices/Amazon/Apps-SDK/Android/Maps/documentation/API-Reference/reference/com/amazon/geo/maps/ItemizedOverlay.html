Class Overview

Base class for a map overlay that contains discrete items. Individual items are represented by the OverlayItem class, and visually indicated by a marker (a drawable). ItemizedOverlay is responsible for:

  • Management of the list of individual overlay items (instances of OverlayItem).
  • Drawing individual item markers on the overlay.
  • Setting bounds for item markers (instances of Drawable.)
  • Handling taps on the overlay or on a specific item.
  • Keeping track of the item that has focus, and dispatching focus events.

Item objects are stored in an item list in the order in which they are created. By default items are drawn in latitude order, north to south. Override getIndexToDraw(int) to determine the order in which items are drawn.

Taps to items on the overlay are managed by overriding the onTap(int) method. Manage taps to the overlay itself (outside of taps) with the onTap(GeoPoint, MapView) method.

This class is not thread-safe.

Public Constructors

public ItemizedOverlay (Drawable defaultMarker)

Creates a new instance of ItemizedOverlay. The defaultMarker argument is used to draw items on the map. You can define different markers on specific items with the setMarker(Drawable) method.

OverlayItem marker drawables can have two states: normal and focused. ItemizedOverlay subclasses should ensure that the bounds have been set on the item markers with the boundCenter(Drawable) or boundCenterBottom(Drawable) methods.

Parameters
defaultMarker the default marker for items on the overlay.

Public Methods

public void draw (Canvas canvas, MapView mapView, boolean shadow)

Draws the markers on the overlay canvas for each Item in the item list. If the populate() method has not been called before this method is invoked, no drawing occurs.

The default marker for this overlay (as defined in the constructor) is used to draw the item markers, unless the Item defines its own custom marker.

The overlay is drawn twice: once for the marker shadows (shadow is true), and once for the markers themselves (shadow is false). Marker shadows are drawn skewed, darkened and anchored to the bottom center of the original marker.

Before items are drawn, they are ranked. The rank of the Item objects in the item list may not be the same as the sequential order of that list. By default, the items are drawn in latitude order, from north to south. Override getIndexToDraw(int) to change the order in which items are drawn.

The Item object that currently has focus is always drawn last so that it appears on top.

Parameters
canvas the Canvas to draw upon.
mapView The MapView that contains this overlay.
shadow If true, draw a shadow layer. If false, draw the markers.

public Item nextFocus (boolean forwards)

Gives focus to the the next (or previous) item in the item list.

If forwards is true, this method returns the next item in the item list. If no item currently has focus, returns the first item in the item list. Returns null if the item that has focus is the last item in the list, or if there are no items in the item list.

If @{code forwards} is false, this method chooses items from the list in reverse, returning the previous item in the item list. Returns null if no item currently has focus, the current focus is on the first item, or if there are no items in the item list.

Parameters
forwards true to choose the next item from the list in increasing order; false for decreasing order.
Returns
  • the next Item object in the item list, or null if there are no subsequent items in the list. If forwards is false, also returns null if no item currently has focus.

public boolean onSnapToItem (int x, int y, Point snapPoint, MapView mapView)

Callback for snapping points to items. Checks if a given x,y point on a MapView is close to a given item (can be snapped), and if so, returns true. If the point can be snapped, the snapPoint parameter contains the point of the item.

By default, a point can be snapped if it is within the bounds of that item's marker drawable. Override hitTest() in your subclass to change how a hit on an item is determined.

Parameters
x X coordinate of the point to test
y Y coordinate of the point to test
snapPoint If the point is snappable, set to the point of the item.
mapView The MapView object requesting the snap
Returns
  • true if the location could be snapped; false otherwise.

protected final void populate ()

Fill the item list with new Item objects. Subclasses should invoke this method each time new items are added to the item list.

This method invokes size() to get the size of the item list, and createItem(int) to create and/or initialize each item in the list. Item-specific behavior is determined by the implementation of createItem(int).

protected void setLastFocusedIndex (int lastFocusedIndex)

Sets the last focused index to the index of the item that currently or most recently had the focus, or -1 if no item has had the focus. Note that this method does not actually set the focus, or change the order of items in the Item list. The last focus is used with nextFocus(boolean) to iterate through the Item objects in the item list.

This property is automatically updated on every focus change.

Parameters
lastFocusedIndex the index of the most recently focused or currently focused item.
