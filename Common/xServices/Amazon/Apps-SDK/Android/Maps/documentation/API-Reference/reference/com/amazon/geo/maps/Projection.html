Projection

com.amazon.geo.maps.Projection

Class Overview

A Projection serves to convert between on-screen pixels (x,y) to real world points (latitude and longitude). A Projection can be obtained from a MapView via getProjection().

Public Methods

public abstract GeoPoint fromPixels (int x, int y)

Given on-screen pixels relative to the top-left of the MapView that provided this projection, this returns a real world point that maps to the given on-screen pixels.

Parameters
x Horizontal pixel count from the top left corner of the MapView.
y Vertical pixel count from the top left corner of the MapView.

public abstract float metersToEquatorPixels (float meters)

Given a distance along the equator (in meters), this calculates the number of horizontal pixels this represents at the current zoomlevel.

In the Mercator projection, as you move further away from the equator the number of pixels for a given distance will increase.

Parameters
meters The number of meters along the equator.
Returns
  • The number of pixels corresponding to the distance measured along the equator, at the current zoom level. The return value may only be approximate.

public abstract Point toPixels (GeoPoint in, Point out)

Returns a set of on-screen pixel co-ordinates given a real world latitude and longitude. The on-screen pixels are relative to the top left of the MapView that produced this projection.

Parameters
in The real world point.
out A Point object that we can update; if no object is provided, a new Point will be created, updated and returned.
Returns
  • Point If out is null, a new Point object is returned with the converted pixels. Otherwise null is returned.
