Method Detail
create()method
public static function create():AmazonPurchase

Initializes the Amazon Purchase Extension. You should check AmazonPurchase.isSupported() first to ensure the platform is Android; you will also want to avoid instantiating the extension on Android builds not destined for the Amazon App Store. AmazonPurchase is a Singleton and you may have only one instance. After creation, you may access the instance through the static property AmazonPurchase.amazonPurchase. Any time after creation, AmazonPurchase may dispatch an AmazonPurchaseEvent.PURCHASES_UPDATED event- this will contain an updated list of the user's purchase history for non-consumable content, in its 'receipts' parameter.

Returns
AmazonPurchase — initialized instance of AmazonPurchase extension interface.

Throws
Error — if AmazonPurchase has already been initialized.

loadItemData()method 
public function loadItemData(itemSkus:Array):void

Retrieve details about a particular item with the given sku. After calling loadItemData(), AmazonPurchase will dispatch one of the following events: AmazonPurchaseEvent.ITEM_DATA_LOADED if the data is loaded successfully. The 'itemDatas' property of the evenet will be a vector array of AmazonItemData objects for each given sku. If any skus are no longer available, they will be in the vector array property 'unavilableSkus'. AmazonPurchaseEvent.ITEM_DATA_FAILED if an error occurs loading the item data.

Parameters

itemSkus:Array — an array of the String skus for the items to load data about


Throws
ArgumentError if — the given parameter itemSkus is not a valid Array of strings

purchaseItem()method 
public function purchaseItem(itemSku:String):void

Initiate a purchase attempt for an item with the given Sku. In response, AmazonPurchase will dispatch one of the following events: AmazonPurchaseEvent.PURCHASE_SUCCEEDED if the purchase is successful. The event's 'receipts' parameter will be a vector array containing an AmazonPurchaseReceipt object for the purchased item. AmazonPurchaseEvent.PURCHASE_FAILED if the purchase was not successful- either due to an error, or the user declining the purchase. AmazonPurchaseEvent.PURCHASE_SKU_INVALID if the given itemSku was not found in the system. AmazonPurchaseEvent.PURCHASE_ALREADY_ENTITLED if the user already owns the particular non-consumable item.

Parameters

itemSku:String — the sku of the item to purchase

PURCHASES_UPDATED Event  
Event Object Type: com.amazon.nativeextensions.android.events.AmazonPurchaseEvent

Dispatched when the status of purchased Items has been updated for the user. This event may be dispatched any time after the SDK is initialized. If any SKUs have been revoked since the last time the extension was initialized, they will be contained in the revokedSkus property. If there are any receipts available for currently updated purchases, they will be contained in the receipts property.

Dispatched After the Amazon Purchase SDK has been initialized by calling AmazonPurchase.create(). The event property 'isSandboxMode' will be TRUE if the app is currently in sandbox/testing mode, or FALSE if the app is currently in production mode (live on the Amazon App Store.)
