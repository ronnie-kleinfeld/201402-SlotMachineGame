Class Overview

This class provides constants that are used only if you are crafting intents manually. Unless you are migrating a codebase from C2DM to ADM, you should use the high-level ADM API via ADM and ADMMessageHandlerBase.

The ADMConstants class contains constants, including error strings and special key strings passed with ADM messages, that are used in both the low-level and high-level APIs.

Registering/Unregistering for ADM with the Low-Level API

Start the registration service (to unregister, use ACTION_UNREGISTER instead:

Intent registrationIntent = new Intent(ADMConstants.LowLevel.ACTION_REGISTER);
PendingIntent myId = PendingIntent.getBroadcast(mContext, 0, new Intent(), 0);
registrationIntent.putExtra(ADMConstants.LowLevel.EXTRA_APPLICATION_PENDING_INTENT, myId);

mContext.startService(registrationIntent);

The service registers your app asynchronously and fires a broadcast back to you. You must register for this broadcast in your manifest, as ADM also sends it if your registration ID needs to be updated: 

 <receiver android:name="<your receiver name>"
 <!-- This permission ensures that only ADM can send you
 registration broadcasts -->
 android:permission="<com.amazon.device.messaging.permission.SEND">
 <intent-filter>
 <action android:name="com.amazon.device.messaging.intent.REGISTRATION"/>
 <category android:name="<your package name>"/>
 </intent-filter>
 </receiver>

In your receiver, handle the registration state change:

if (intent.hasExtra(ADMConstants.LowLevel.EXTRA_ERROR))
{
    // An error occurred. ADM has already retried the request a few times, so this should be considered
    // fatal and handled accordingly.
    handleRegistrationError(intent.getStringExtra(ADMConstants.LowLevel.EXTRA_ERROR));
}
else if (intent.hasExtra(ADMConstants.LowLevel.EXTRA_UNREGISTERED))
{
    // Notify your server that messages can no longer be sent to this device
    handleUnregister(intent);
}
else if (intent.hasExtra(ADMConstants.LowLevel.EXTRA_REGISTRATION_ID))
{
    // Notify your server of the new registration ID for this device
    handleNewRegistrationID(intent);
}
else
{
    // You should never get anything else, but always code defensively...
    handleRegistrationError(ADMConstants.ERROR_SERVICE_NOT_AVAILABLE);
}

Receiving Messages

Register a broadcast receiver to receive the messages (it can be the same one you registered for handling registrations):

 <receiver android:name="<your receiver name>"
 <!-- This permission ensures that only ADM can send you messages -->
 android:permission="<com.amazon.device.messaging.permission.SEND">
 <intent-filter>
 <action android:name="com.amazon.device.messaging.intent.RECEIVE"/>
 <category android:name="<your package name>"/>
 </intent-filter>
 </receiver>

In your receiver, handle the message:

 public void onReceive(final Context context, final Intent intent)
 {
 Bundle payload = intent.getExtras();

 // The extras are the key/value pairs your server provided to the ADM server
 }
